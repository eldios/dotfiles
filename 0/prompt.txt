# avante.nvim vs codecompanion.nvim — Neovim community sentiment, 2026

Most "2026" writeups are blog/aggregator pieces that recycle the same GitHub Discussion (`codecompanion.nvim` #1209). The richest primary sentiment is in that discussion, the LazyVim discussion, Lobsters/HN threads, and the history-extension repo. ## 1. Sentiment summary per plugin

### avante.nvim — "Cursor inside Neovim"
**Praised for:** the diff-first sidebar workflow (select → describe → review diff → accept/reject with one keystroke), broad multi-provider support (Claude, OpenAI, Gemini, Copilot, Ollama, plus ACP agents: claude-code, gemini-cli, codex, goose, kimi-cli), per-project `avante.md` steering (the CLAUDE.md parallel), and well-handled code diffs. Users doing non-trivial refactors specifically prefer it because the explicit review step matters (Samuel Lawrentz blog, 2026-03-26).

**Complained about (recurring, from users who left):**
- "too loaded, heavy and hard to use now. There is just no documentation" — *pratyushmittal*, CC Discussion #1209.
- "buggy and it doesn't work with copilot," plus unresolved bug reports — *TheLazyCat00*, #1209.
- Context overflow causing freezing, slow issue resolution, confusing release notes, and repeated build-artifact re-downloads on update — *gitsang*, #1209.
- Build friction: `make BUILD_FROM_SOURCE=true` trips up new users (avante GitHub issues).
- Breaking changes tracking fast-moving model APIs (e.g. the Opus 4.7 `temperature` deprecation 400-error, issue #3022).
- Keybinding conflicts with LazyVim defaults (LazyVim Discussion #4402).

**Consensus read:** avante is the popular, feature-rich, higher-velocity option — and that velocity is exactly the double-edged sword. The core knock is *internal complexity / instability / thin docs*, not the UX. Its sidebar+diff UX is consistently the thing people *praise*, even detractors.

### codecompanion.nvim — "feels like Neovim, not Cursor"
**Praised for:** documentation quality and simplicity ("love CodeCompanion for its documentation and simplicity" — *pratyushmittal*), a clean codebase that's easy to modify/extend, clear release notes flagging breaking changes, deep native integration (chat buffers in splits, `@lsp`/`@buffers` variables, slash commands, prompt library, GitHub Models free tier), and a cautious maintainer who ships fewer but more stable features.

**Complained about:**
- Cautious pace means "might take longer to solve problems" (*gitsang*, #1209).
- Historically weaker at *agentic* coding: "works well for interactive chat but falls short for agentic coding… limited to pre-configured 'workflows'" (HN #44839311, ~2025).
- Chat persistence historically clunky: "CodeCompanionSave/CodeCompanionLoad requires quite the dance" (Discussion #139/#652).

**Consensus read:** the "Neovim-native, stable, well-documented, extensible" choice. The recurring reservation is agentic depth and (historically) chat persistence — both of which have been actively closing in late-2025/2026.

## 2. Who should pick which (community synthesis)

- **Pick avante** if you want a Cursor-like, diff-first review workflow for non-trivial refactors, the widest provider/ACP-agent menu out of the box, and per-project `avante.md` steering — and you can tolerate faster churn, occasional bugs, thinner docs, and a source build.
- **Pick codecompanion** if you value Neovim-native feel, stability, excellent docs, an easily-hackable/extensible codebase, and tight LSP/buffer context — and you're doing more chat/in-flow work than heavy autonomous agentic edits.
- **The most-repeated actual advice: install both.** They don't conflict; bind to different keymaps and use avante for big diffs, codecompanion for in-flow questions (Samuel Lawrentz 2026; *gitsang* #1209). Several heavy users also run an external CLI (Claude Code in a tmux pane) for truly agentic work regardless of plugin (HN #44839311, Lobsters).

**Direction of migration:** the visible movement in the primary sources is **Avante → CodeCompanion** (pratyushmittal, TheLazyCat00, caesar-ipl in #1209), driven by avante's complexity/bugs/doc gaps — not by feature envy. I found **no** documented CodeCompanion → Avante migrations, though this is partly a sampling artifact (the richest thread lives on the CodeCompanion repo, so it over-samples people who chose CC). Treat "everyone is leaving avante" as a **loud, self-selected minority**, not proven broad consensus: avante still has ~2.7x the stars and heavy ongoing use.

## 3. The chat UI / multi-session history / UX-polish question (your specific concern)

Your recollection — *you prefer avante's UX; codecompanion historically lacked chat + multi-history* — was **accurate for 2024–early 2025, but is now contradicted by 2026 reality on the history point.**

- **Chat multi-session history — codecompanion has caught up, and arguably surpassed avante here.**
  - Native: CodeCompanion now supports **ACP session resume** — `/resume` in a fresh chat buffer calls `session/list` → `session/load` to restore prior conversations across restarts (docs last updated 2026-06-30; ACP Discussion #2313).
  - Extension: **`ravitemer/codecompanion-history.nvim`** (created 2025-05-03, 188 stars) gives full multi-session management — save/browse/restore, smart auto-generated titles, project-aware filtering, chat duplication, and a vector-search summary system over past chats (telescope/snacks/fzf-lua pickers). This is genuinely richer multi-history than avante ships. It preserves messages, tools, schemas, and references.
  - Caveat: this is a *third-party extension*, not core, and its last push was 2026-01-22 (stable but less active); there's a known open bug (issue #63) about message history not passed to ACP adapters on restore. Avante's own persisted, browsable multi-session history is comparatively thin — avante is centered on the live sidebar/diff loop, not a chat archive.

- **Chat UI quality / conversations — codecompanion is strong; this is arguably its home turf.** Its chat-buffer-in-a-split model, slash commands, `@` context variables, and prompt library are consistently what people praise (#1209, Samuel Lawrentz). TheLazyCat00's reason for leaving avante was literally that CC "supports conversations (not just buffer editing)." So the framing "codecompanion lacked chat" is now inverted — chat is its strength.

- **Overall UX polish — this is where your preference still holds up.** "Polish" splits two ways in the sources:
  - *Interaction polish for edits*: avante's one-keystroke diff accept/reject sidebar is repeatedly called the more Cursor-like, satisfying flow for applying changes. If "UX polish" means the inline-edit/diff experience, avante still wins in community sentiment.
  - *Docs / stability / coherence polish*: codecompanion is judged the more polished, less janky, better-documented product. Avante's UX is praised but its *reliability and docs* are the top complaint.

**Bottom line for you:** the specific gap you remember (no chat, no multi-history in codecompanion) has been closed as of 2026 — via native ACP resume plus `codecompanion-history.nvim`. Your preference for avante's *diff/sidebar interaction feel* is still shared by the community; but if the deciding factor is "multi-session chat history + chat UI," codecompanion is now at least on par and by several accounts ahead — with the asterisk that its best history story depends on a third-party extension.
