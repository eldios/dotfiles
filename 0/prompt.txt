### How each app consumes the theme (verified against docs)

- **rio**: Rio's config key `theme = "&lt;name&gt;"` loads `~/.config/rio/themes/&lt;name&gt;.toml`, a TOML file with a `[colors]` section (`background`, `foreground`, `cursor`, `selection-background`, `selection-foreground`, ansi `black`..`white`, `light-black`..`light-white`; hex strings). Source: [rioterm.com/docs/config](https://rioterm.com/docs/config). Rio applies defaults for any missing field, so an empty placeholder is safe.
- **waveterm**: Wave reads terminal themes from `~/.config/waveterm/termthemes.json` AND `~/.config/waveterm/termthemes/*.json` (merged, keyed by theme id); `term:theme` in `settings.json` selects one by key. Theme keys: `display:name`, `display:order`, `background`, `foreground`, `cursor`, `selectionBackground`, `cmdtext`, `black`..`white`, `gray`, `brightBlack`..`brightWhite`. Sources: [waveterm default termthemes.json](https://github.com/wavetermdev/waveterm/blob/main/pkg/wconfig/defaultconfig/termthemes.json), [docs.waveterm.dev/customization](https://docs.waveterm.dev/customization). Wave falls back to a built-in theme if the named key is absent, so an empty `{}` placeholder is safe. Both apps CAN read external theme files cleanly — nothing had to be left inline.

Since neither app has an "include arbitrary path" directive (unlike ghostty/kitty), both consume the omarchy-rendered file via a `config.lib.file.mkOutOfStoreSymlink` from their theme dir into `~/.config/omarchy/current/theme/`, so runtime theme swaps are picked up without a rebuild.
