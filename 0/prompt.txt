## Key findings on your specific questions

**1. blink.cmp** — Yes, it is now the de-facto modern completion engine. LazyVim ships `blink.cmp` as its **default** completion (via the `coding.blink` extra); `nvim-cmp` is now the opt-in alternative extra, and exactly one is active at a time. Latest stable v1.10.2 (2026-04). No notable stability issues surfaced; recent releases are bugfix/polish. **KEEP.**

**3. dressing.nvim** — Confirmed **ARCHIVED**. README top notice, verbatim:
&gt; "This plugin is archived! It still works, but I recommend that you use snacks.nvim instead for your `vim.ui.*` interfaces."
Since you almost certainly already have snacks.nvim (LazyVim core), just enable `snacks.input`/`snacks.select` and drop dressing. **REPLACE.**

**4. render-markdown vs markview** — Both are actively developed in 2026. render-markdown.nvim (v8.13.0, 2026-06-18) is the one broadly wired into AI-chat plugins (avante, codecompanion, copilot-chat) and is what LazyVim's AI extras pull in. markview.nvim (v28.3.0, 2026-05-16) is equally maintained but a heavier, more feature-broad renderer. **No reason to switch off render-markdown.**

**6. nvim-cmp redundancy (important)** — nvim-cmp is not archived and still maintained, but if blink.cmp is your main engine, having nvim-cmp loaded **only** as an avante dependency is redundant. avante can use blink directly via **Kaiser-Yang/blink-cmp-avante** (add it as a blink dependency and add `avante` to blink sources), or via `blink.compat`. Recommendation: switch avante to the blink-cmp-avante provider and drop the nvim-cmp dependency, so you don't run two completion frameworks. **REMOVE-redundant.**

**7. mini.pick / nvim-web-devicons** — Both current and fine. Note the org rename: mini.pick now lives under `nvim-mini/mini.nvim` (was `echasnovski`). The only "modern shift" worth knowing: LazyVim's default picker is now **snacks.picker**, and **mini.icons** is a lighter icon provider many configs adopt — but neither is a required migration; your current choices are fully supported.
